# KeyRemapper — 要件・設計メモ

## 概要
Keychron Launcher ライクな GUI でキーボードのキーマップを編集し、
JSON 設定ファイルとして入出力、AutoHotkey スクリプトを生成する。

## 構成
- `index.html` — メインUI（HTML/CSS/JS、ブラウザで直接動作）
- `css/style.css` — ダークテーマ
- `js/app.js` — 全ロジック（キーボード描画・レイヤー管理・リマップ・JSON入出力・AHK生成）

## 機能一覧

### キーボード配列の選択
- ANSI 104（フルサイズ）
- JIS 109（日本語フルサイズ）
- TKL/60% は削除済み

### レイヤー管理
- レイヤーの追加・削除・リネーム
- アクティブレイヤーの切替
- デフォルトで2レイヤー（Default + Layer 1）
- 透過キー表示のON/OFF切替

### キー割当（リマップ）
**アクション種類:**
| 種類 | 説明 | JSON形式 |
|------|------|----------|
| キー入力 | 通常のキー→キー割当 | `"a": "b"` |
| ModTap | タップと長押しで別機能 | `"a": "modtap:a:lctrl"` |
| レイヤー切替(MO) | 押下中のみレイヤー切替 | `"caps": "layer:1"` |
| 透過 | 下位レイヤーの設定を継承 | `"a": "transparent"` |
| 無効 | キーを無効化 | `"a": "none"` |

### JSON入出力
- File API でブラウザから直接ファイルを読込/保存
- フォーマット:
```json
{
  "version": 1,
  "keyboard": "ansi104",
  "layers": [
    { "name": "Default", "mappings": { "a": "b" } }
  ]
}
```

### AutoHotkey スクリプト生成
- レイヤー状態を `CurrentLayer` グローバル変数で管理
- `#If (CurrentLayer = n)` でレイヤー別マッピング
- ModTap/MO/通常マッピングは全て `$*` プリフィクス統一（AHK特異性競合防止）
  - `$a::`（非ワイルドカード）と `$*a::`（ワイルドカード）が混在するとAHKが特異性の高い方を優先し、`#If` falseでもフォールバックしない問題 → `$*`で統一して解決
- `#MaxThreadsPerHotkey, 20` + ガード変数（`_busy_<key>`）で長押し時のオートリピート連打を防止
  - `$*`はオートリピートも含むため、スレッドキューイングで短キー連打が発生 → `_busy`ガードで初回のみ処理
- ModTap 長押しMO: 即時レイヤー切替 + 300msタイマーでタップ検出
- ModTap 長押しキー: KeyWait T0.2（200ms）でタップ/ホールド判定
- MO: `$*KeyName` + `KeyWait`（解放までブロック） + `_MO_count`/`_MO_base`で重畳対応
- レイヤー切替表示（オプション）: `_CheckLayer` 200msタイマー + ToolTip 3秒
- Ctrl+Shift+D でデバッグ表示（現在レイヤーをToolTip）

## キーID命名規則
- 英字: `a` `b` `c` ...
- 数字: `1` `2` `3` ...
- 記号: `grave` `minus` `equal` `comma` `dot` `slash` `semicolon` `quote` `lbracket` `rbracket` `backslash`
- 操作: `esc` `tab` `caps` `backspace` `enter` `space` `delete` `insert` `home` `end` `pgup` `pgdn` `prtsc` `scroll` `pause` `menu`
- 修飾: `lshift` `rshift` `lctrl` `rctrl` `lalt` `ralt` `lwin` `rwin`
- Fn: `f1` `f2` ... `f12`
- 矢印: `up` `down` `left` `right`
- テンキー: `n0`-`n9` `ndot` `nplus` `nminus` `nmult` `ndiv` `nenter` `nlock`
- JIS固有: `hankaku` `yen` `caret` `at_mark` `lbracket_jis` `colon_jis` `rbracket_jis` `ro` `muhenkan` `henkan` `kana`

## ModTap AHK生成ロジック（現在） — `$*` + ガード変数 + `#MaxThreadsPerHotkey,20`
```autohotkey
   ; 共通: #MaxThreadsPerHotkey, 20 でオートリピートスレッド即時発行
; ガード変数 _busy_<key> で初回以外を弾く
; $key up:: でガードクリア（キー解放時）

; ModTap 長押しMO: 即時レイヤー + 300msタップ検出
$*KeyName::
  global _busy_KeyName
  if (_busy_KeyName)
    return
  _busy_KeyName := true
  _MT_key_held := false
  _MT_anykey := 0              ; 他キー入力検出用
  _MO_count++
  if (_MO_count = 1)
    _MO_base := CurrentLayer
  CurrentLayer := N
  SetTimer, _MT_key_chk, -300
  KeyWait, KeyName
  SetTimer, _MT_key_chk, Off
  _MO_count--
  if (_MO_count = 0)
    CurrentLayer := _MO_base
  if (!_MT_key_held && !_MT_anykey) {
    SendInput TapKey           ; 300ms以内UP + 他キーなし → 単押し
  }
  ; _busy_* はここでクリアしない → $key up:: 任せ
return
_MT_key_chk:
  _MT_key_held := true         ; 300ms経過 → 長押し確定
return

; ModTap 長押しキー: 200ms KeyWait
$*KeyName::
  global _busy_KeyName
  if (_busy_KeyName)
    return
  _busy_KeyName := true
  _MT_anykey := 1
  KeyWait, KeyName, T0.2
  if (ErrorLevel) {
    Send {Blind}{HoldKey DownR}
    KeyWait, KeyName
    Send {Blind}{HoldKey Up}
  } else {
    SendInput TapKey
  }
  _busy_KeyName := false
return

; レイヤー切替 (MO) — $* + KeyWait + _MO_count/_MO_base
$*KeyName::
  global _busy_KeyName
  if (_busy_KeyName)
    return
  _busy_KeyName := true
  _MO_count++
  if (_MO_count = 1)
    _MO_base := CurrentLayer
  CurrentLayer := N
  _MT_anykey := 1
  KeyWait, KeyName
  _MO_count--
  if (_MO_count = 0)
    CurrentLayer := _MO_base
  _busy_KeyName := false
return

; 通常マッピング（#If内）
$*KeyName::
  _MT_anykey := 1
  SendInput {Blind}{Target}
return

; 修飾キー割当（#If内）
$KeyName::
  _MT_anykey := 1
  Send {Blind}{Mod DownR}
return
$KeyName up::
  Send {Blind}{Mod Up}
return
```

## JIS 109 配列 メモ
- 数字行: 半/全 | 1-0 | - | ^(SC00D) | ¥(SC02B) | BS(1.5)
- Tab行: Tab(1.5) | Q-P | @(SC01A) | [(SC01B) | gap(0.5) | Enter(1, 縦2段)
- Home行: Caps(1.75) | A-L | ;(SC027) | :(') | ](1.0) | gap(0.5)
- Bottom行: LShift(2.25) | Z-/ | ro(SC073) | RShift(1.25)
- JIS専用キー: hankaku, caret(sc00d), yen(\\), at_mark(SC01A),
  lbracket_jis(]), rbracket_jis... wait, `]` is actually at SC027 on JIS tab row,
  but SC027 on JIS home row is `;`. Need scancode verification.

## 既知の制約
- (解消) AHK特異性問題: `$a::` vs `$*a::`の混在で、`$a::`の#Ifがfalseでもフォールバックせず → `$*`統一で解決（2026/07/06）
- (解消) `$*`オートリピート連打: `$*`はオートリピートも含むため、スレッドキューイングで短キー連打が発生 → `#MaxThreadsPerHotkey,20` + `_busy`ガード変数で解決（2026/07/06）
- (解消) JIS Enter の縦長表示: CSS margin-bottom 方式で 2行スパン対応済み（Enter幅1.75u）
- `;` キーは AHK のコメント記号と競合するため `sc027` で代用
- FireFox は `msSaveOrOpenBlob` 非対応、保存のみ可能
- IE/Edge(EdgeHTML) のみ「保存して実行」が動作
